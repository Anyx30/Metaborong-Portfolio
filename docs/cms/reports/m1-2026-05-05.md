## Defects (severity-ordered)

### HIGH

1. **`@vercel/postgres` cannot connect to a vanilla local Postgres;
   the dispatch's "Easiest" setup recipe silently does not work.**
   The Tester dispatch instructs: *`docker run --rm -d --name
   mb-test-pg -p 5432:5432 -e POSTGRES_PASSWORD=test postgres:16` ‚Ä¶
   then `POSTGRES_URL=postgres://postgres:test@localhost:5432/postgres`*.
   This recipe applies migrations correctly (drizzle-kit uses
   `pg`/libpq) but every runtime route handler that touches the DB
   crashes with `ECONNREFUSED` on `wss://localhost/v2`, because
   `db/client.ts` uses `drizzle-orm/vercel-postgres` over
   `@vercel/postgres`, which is a Neon-WebSocket transport. The
   driver only talks to a Neon (or Neon-compatible) endpoint;
   plain Postgres on TCP is not supported.

   **Impact on this dispatch:** the entire B matrix (login happy / 401
   / 422 / 429 / first-CSRF) and the credentialed half of E and F
   and all of H cannot be verified end-to-end at runtime. Implementation
   is verified by the 70/70 BE vitest suite via pg-mem; the gap is
   purely the dispatch's "spin up a real DB and curl it" workflow.

   **Notes:**
   - BE handoff ¬ß8 already flags `@vercel/postgres` as deprecated and
     suggests migrating to `@neondatabase/serverless` in M8 ‚Äî but that
     SDK swap *also* needs a Neon endpoint or a Neon HTTP/WSS proxy
     (e.g. `ghcr.io/timowilhelm/local-neon-http-proxy`); it doesn't
     by itself unblock vanilla Postgres.
   - PRD ¬ß6.1 explicitly chose Vercel Postgres / Neon. So in
     production this is correct. The defect is purely the *local-dev
     story*, which neither the BE handoff ¬ß7 nor the Tester dispatch
     documents.

   **Suggested fix (one of):**
   - (a) Document a Neon dev-branch URL or a local Neon HTTP/WSS
     proxy as the canonical local-dev setup. Update the Tester
     dispatch and BE handoff ¬ß7 prerequisites.
   - (b) Make `db/client.ts` switch driver based on `POSTGRES_URL`
     hostname: use `pg`/`drizzle-orm/node-postgres` for `localhost` /
     `127.0.0.1`, `vercel-postgres` for Neon hosts. Two-line change
     plus a new dev dep already-installed (`pg` / `pg-mem` are in
     devDeps).
   - (c) Adopt `@neondatabase/serverless` + bundle a docker-compose
     that ships the `local-neon-http-proxy` alongside Postgres so the
     dispatch's recipe just works.

2. **FE branch is missing the `test:e2e` package.json script that

### LOW / NIT

6. **NIT: 307 vs 302.** The dispatch describes the gate as "302 to
   `/admin/login?next=‚Ä¶`"; Next.js's `redirect()` emits 307. Both
   are temporary redirects per RFC 9110; 307 is the modern correct
   choice (preserves method on redirect, which matters when
   redirected POST ‚Üí GET behavior would otherwise be ambiguous).
   Documentation nit, not a defect.

7. **NIT: noindex meta has a space.** `/admin/login` emits
   `content="noindex, nofollow"` (with one ASCII space); the
   dispatch quoted `"noindex,nofollow"` (no space). Both are valid
   per HTML 5 / robots-meta spec; Google, Bing, and OpenAI all
   normalize. Cosmetic only.

8. **Auth-gate cookie name appears with lowercase `lax`** in the
   `Set-Cookie` clearing headers (`SameSite=lax`); contract ¬ß2.1
   uses `SameSite=Lax`. Per RFC 6265 case-insensitive. Cosmetic.

9. **NIT: `next dev` warns about multiple lockfiles** because
   `~/Documents/Github/package-lock.json` exists in a parent
   directory. Not a CMS defect; a workspace-root inference quirk.
   Low signal; harmless.
