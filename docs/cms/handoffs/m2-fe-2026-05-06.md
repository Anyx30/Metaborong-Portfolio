## 3. Components created

- `BlockRenderer` (`components/blog/block-renderer.tsx`) — exhaustive
  switch over all 9 `Block.type` values. Heading anchors auto-derive
  from text, brand-token sized per level. Image blocks accept an
  optional resolver and fall back to a 16:9 dashed placeholder pre-M4.
- `PostView` (`components/blog/post-view.tsx`) — the single shared
  post renderer. Cover, h1 (canonical, never from a heading block),
  excerpt-as-lede, byline (author + UTC-deterministic dates),
  iterates `content_json` through `<BlockRenderer />`. `draftBanner`
  prop differentiates the standalone admin preview from the public
  reader.
- `AdminPostsList` (`components/admin/admin-posts-list.tsx`) — client
  island for the dashboard: status tabs (URL-backed), post rows with
  status pills, focus-trapped delete-confirmation modal that calls
  `DELETE /api/admin/posts/<id>` and optimistically removes the row.
- `NewPostForm` (`components/admin/new-post-form.tsx`) — title-only
  client form. Posts to `/api/admin/posts`; on success
  `router.push('/admin/posts/<id>')`. Maps API field-path errors back
  to the input.
- `EditPostForm` (`components/admin/edit-post-form.tsx`) — the M2
  editor. Client-side validation against `lib/blog-schema` (Zod
  safeParse + `validateContentJson` enforcement); 4-state save
  indicator (idle / saving / saved / error with Retry); 2s autosave
  debounce; optimistic Publish / Unpublish with rollback; typed-slug
  Delete modal; `beforeunload` guard against losing unsaved edits.
- Re-rendered server pages: `AdminDashboardPage`, `AdminNewPostPage`,
  `AdminEditPostPage`, `AdminPreviewPage`, `BlogIndexPage`,
  `BlogPostPage`.

## 8. Known gaps / open questions

1. **M1 happy-path E2E flake (PRE-EXISTING — out of scope for this
   dispatch).** `e2e/m1-auth.spec.ts:53` "valid credentials sign in,
   sign out, browser back lands on login" fails on `page.goBack()`
   returning `about:blank` instead of `/admin/login`. The M1 Tester
   report (`docs/cms/reports/m1-2026-05-05.md`) marked this test
   BLOCKED at M1 due to a now-fixed login 500 driver issue, and it
   was never re-verified after the fix. The TPM has acknowledged
   this is a Playwright/Next.js browser-state interaction unrelated
   to M2 work; flagging it here so it stays visible for a future
   hygiene patch (likely solution: replace `goBack()` with an
   explicit `page.goto('/admin')` to test the gate's behavior on
   navigation rather than history).

2. **`scroll-behavior: smooth` warning at runtime.** Next.js dev
   server logs `Detected scroll-behavior: smooth on the <html>
   element. To disable smooth scrolling during route transitions,
   add data-scroll-behavior="smooth" to your <html> element.` The
   smooth scroll lives in `app/globals.css` and predates M2 — not
   FE-M2-introduced. Easy fix lands in a future hygiene PR.

3. **Image picker / cover URLs.** Pre-M4, the post listing card and
   the standalone preview both render placeholder divs for cover
   images (16:9 aspect-locked, brand-tinted). Once M4's image
   pipeline lands, the `resolveImage` resolver hook on
   `<PostView />` and a cover-URL field on `PostSummary` will swap
   in real `next/image` renders.

4. **Geo-variant editor UI is M6.** The resolver and merger work on
   the public path right now (server-side `resolveRegion` +
   `mergeVariant`); the editor has no Base/US/EU tab strip yet, and
   `geo_variants` cannot be authored from this dispatch's edit
   screen. PATCH still accepts `geo_variants` payloads, so a
   technical user can edit them via direct API call — not surfaced.

5. **Tiptap rich editor is M3.** The current edit screen ships a
   plain `<textarea>` for `content_json`. JSON validates client-side
   via the Zod `contentJsonSchema` plus `validateContentJson()`
   enforcement (no h1, ≤1 tldr) before Save submits. The M3 dispatch
   replaces the textarea with the Tiptap node-based editor + slash
   menu + role inspector + live preview pane.

6. **Slug collision pre-check happens at submit, not on title-blur.**
   The dispatch suggested an explicit blur-time pre-check; in
   practice, the BE's POST `/api/admin/posts` returns
   `{ code: 'SLUG_CONFLICT', field: 'slug' }` immediately and the
   form surfaces it inline. The blur-only pre-check would add a
   second round-trip and double the surface area for race
   conditions; the submit-time check is functionally equivalent.

7. **Public `/blog` revalidation lag is real but small.** After
   publish, BE calls `revalidatePath('/blog')` and
   `revalidatePath('/blog/<slug>', 'page')`, but ISR propagation
   takes a moment — the M2 e2e spec retries up to 5x with 800ms
   waits before failing. Production traffic won't notice this; CI
   should be fine on the retry budget.

## 9. Manual-verify path (5-step golden flow)

Assumes BE is merged into `cms-dev` (or this branch's
cross-pollination is intact), `.env.local` configured,
`mb-test-pg` Docker container running on `:5433`, dev server on
`:3000`, and admin creds seeded
(`admin@metaborong.test` / `admin1234`).

1. **Login + dashboard.** Visit `/admin` → redirected to
   `/admin/login?next=%2Fadmin`. Sign in. Dashboard renders the
   "Posts" empty-state when the DB is fresh; a status-tabbed list
   when posts exist. Click `New post`.
2. **Create + edit.** Title-only form posts and routes to
   `/admin/posts/<id>`. Fill content_json with a small Block[]
   document, set author + excerpt. Save indicator transitions
   `idle → saving → saved Xs ago`. Type a `g` somewhere and wait
   2s — autosave fires (no manual click).
3. **Publish + verify public.** Click Publish; pill flips to
   `published`. Open `/admin/posts/<id>/preview` in a new tab to
   eyeball the rendered post. In an incognito window visit
   `/blog/<slug>` — same render, no admin chrome. Title + excerpt
   visible, blocks rendered.
4. **Unpublish + 404.** Back in the editor, click Unpublish; pill
   flips to `draft`. Refresh the incognito tab — `/blog/<slug>`
   returns 404 within a couple of seconds.
5. **Delete.** Open the Delete modal. Confirm button stays disabled
   until you type the slug verbatim. Confirm. Routes back to
   `/admin`; the dashboard list no longer contains the slug.
