## 4. Edge cases handled

Mapped to "EDGE CASES YOU OWN" in the dispatch prompt:

| Case | Handling |
|------|----------|
| Empty catalog (no published posts) | `/llms.txt` emits a valid index with `# Metaborong` / `> description` / `## Posts` / `- No posts yet.`. `/llms-full.txt` emits header only with no separator and no post blocks. Both return 200, never 404. |
| Markdown special chars in titles | `/llms.txt` bullet lines escape `\`, `[`, `]`, `(`, `)` so `[title](url)` never breaks. Tested with `Bears [in] a (forest)`. |
| Markdown special chars in `/llms-full.txt` titles | Heading lines (`### …`) only need newline flattening — markdown's heading syntax doesn't reserve `[]()`. We flatten internal whitespace so an embedded newline can't break the post-block boundary. |
| Author / meta lines in `/llms-full.txt` | Same whitespace flattening. The line is a "Key: value" pair so a stray newline in `author_name` would otherwise look like a new field. |
| Per-post markdown body that contains its own H1 | Picked dispatch option (b): emit `### Post Title` then the body verbatim. LLM crawlers consume plain text and don't care about heading nesting. Heading demotion (h1→h2, h2→h3, …) flagged as v1.6 polish — see §7.1. |
| ISO 8601 date for `Published:` line | `toIsoDate(iso)` slices the upstream ISO string at index 10 → `YYYY-MM-DD`. Locale-independent. |
| Encoding | Both routes set `Content-Type: text/plain; charset=utf-8` explicitly. UTF-8 round-trips through `Response`'s default body serializer. |
| 5 MB size guard on `/llms-full.txt` | Bundle is assembled, then `Buffer.byteLength(blob, 'utf8')` measured. If > 5 MB, a `> Note: post catalog exceeds 5 MB; consider per-post fetching via /blog/<slug>/raw.md` block is prepended above `## Posts`. We never truncate; the spec is silent and clipping would silently drop posts. |
| Geo-variant suppression | The helper returns the un-merged base-region row. `/llms.txt` and `/llms-full.txt` are region-neutral; per-region content is reachable via `/blog/<slug>` with the request-time geo header. |

## 7. Known gaps / open questions

1. **Heading hierarchy in `/llms-full.txt`.** Per dispatch we picked option (b)
   — emit `### Post Title` and let post bodies that start with `# ` or `## `
   sit "above" the post's H3. LLM crawlers don't care, but if a future v1.6
   pass adds a polish step, demote body headings by one level (h1→h2, h2→h3,
   …) inside `blocksToMarkdown` via an opt-in `headingShift: 1` option. Don't
   re-implement the serializer; thread the option through.
2. **No `<link rel="alternate" type="text/markdown" href=".../raw.md">` header
   on `/blog/[slug]`.** The dispatch references "the alternate-link header"
   in the lead but lists only the index files in scope. The per-post
   alternate link is a small FE / `app/blog/[slug]/page.tsx` change (set
   `alternates.types['text/markdown']` in the page-level `Metadata`); flagged
   here so the FE dispatch (or a follow-up M9-GEO-FE pass) picks it up.
3. **No pagination / cap on the catalog.** Both routes load every published
   post in one query. Fine for v1.5 (catalog is small); past ~1k posts the
   helper should grow a `limit` parameter and the routes should split into
   paginated `/llms.txt?page=N` (the spec doesn't define pagination, so this
   would need an llmstxt.org-community decision first).
4. **5 MB warning is informational.** Emitting it does not change behavior —
   the bundle still ships in full. If LLM crawlers report truncation, swap
   the warning for a hard cut at the warning point with a `> truncated; see
   /blog/<slug>/raw.md for the rest` trailer.
5. **The 5 MB test seeds a single very large paragraph (~7.2 MB).** It runs
   in ~6 s under pg-mem and is given a 20 s timeout. If `npm test` flakiness
   is reported on this case in CI, drop the seed size to 5.5 MB or move the
   warning trigger threshold check into a unit test that calls a separate
   helper (extract `assembleBundle()` / `shouldWarnAtSize()`).
