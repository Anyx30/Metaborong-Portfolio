## 4. Edge cases handled

| Case | Handling |
|------|----------|
| `post.canonical_url` set to an external syndication target | The markdown alternate is constructed from `post.slug` against the metaborong.com origin, NOT from `canonical_url`. Concatenating `raw.md` to an arbitrary external URL would emit a broken alternate (and the raw.md route only exists on our domain anyway). |
| Trailing slash on default canonical | The default canonical (`https://www.metaborong.com/blog/<slug>/`) ends with `/`, so a naive `${canonical}raw.md` would also work — but we kept the dedicated `markdownAlternate` variable so the safe construction survives if the canonical convention changes. |
| Relative `/llms.txt` href on the homepage | `metadataBase` (set in `app/layout.tsx` to `https://www.metaborong.com`) absolutizes the relative path at render time. Confirmed in `view-source` of the dev server: `<link rel="alternate" type="text/plain" href="https://www.metaborong.com/llms.txt"/>`. |
| LLMs.txt link semantics in `/blog` header | aria-label is `"LLM-readable index of all posts"` so the screen-reader experience doesn't depend on knowing what `.txt` means. The visible label is still the literal `LLMs.txt` for humans who recognize the convention. |
| Empty post catalog on `/llms-full.txt` e2e | The `/llms-full.txt` test is gated on `postId` so it only runs when the seed step published a post. /llms.txt's "## Posts" check works empty too (the BE always emits the header). |

## 7. E2E spec (`e2e/m9-geo.spec.ts`)

Seven serial steps inside one describe block, all gated on `hasCreds`:

1. **Seed + publish.** Mints a post via `/admin/posts/new` then PATCHes
   minimal `content_json` (heading + paragraph) so `/llms-full.txt`
   has a "### " block to assert against. Same pattern as M5/M9-AEO.
2. **Homepage `<head>` alternate.** `page.locator('link[rel="alternate"][type="text/plain"]')`
   with `toHaveCount(1)` and `toHaveAttribute('href', /\/llms\.txt$/)`.
   Regex suffix-matches so the assertion survives whether the URL is
   absolutized or relative.
3. **Post page `<head>` alternate.** Same shape, `type="text/markdown"`,
   href ends with `/blog/<slug>/raw.md`. Includes the same 5-attempt
   retry loop M5 uses to defeat ISR cold-start delays.
4. **`/llms.txt` content.** Asserts `text/plain` content-type and
   the `## Posts` substring.
5. **`/llms-full.txt` content.** Asserts content-type and at least one
   `^### ` line (multi-line regex). Skipped when seed step didn't run.
6. **`/blog` listing link.** `getByRole('link', { name: /llm-readable index/i })`
   visible and `href === '/llms.txt'`. Also asserts the RSS link is
   visible — both proves the LLMs.txt link sits in the same header.
7. **Cleanup.** Re-login, unpublish, delete via the admin UI.

`afterAll` re-logs-in and DELETEs the seeded post via API even on
mid-suite failure, so the dev DB doesn't leak rows between runs.

## 9. Known gaps / open questions

1. **No deduplication of the LLMs.txt link across the site.** Today
   it lives only on `/blog`. The dispatch chose `/blog` because
   that's the existing RSS surface; if a future change adds a
   sitewide footer the link could move there too. Out of scope for
   M9-GEO.
2. **Speakable / Markdown alternate co-existence.** A post page now
   carries (a) up to 5 JSON-LD scripts from M5-core + M9-AEO and
   (b) the new `<link rel="alternate" type="text/markdown">`. The
   alternate is `<head>`-level and JSON-LD is `<body>`-level
   (Next emits the script tags inside the route's React subtree,
   not the metadata head), so there is no ordering or framing
   conflict. Mentioned here so it's explicit for Tester.
3. **Markdown alternate is unconditional.** Even if a post never
   exercised the raw.md route in M5-core (e.g. a future migration
   that scopes raw.md to a flag), the alternate would still be
   advertised. Today the raw.md route exists for every published
   post via `app/blog/[slug]/raw.md/route.ts`, so there's no gap.
4. **`<link>` alternates not visible to RSC streaming clients.**
   Next renders `<head>` from the Metadata API at the route boundary
   so SSR + RSC clients both see the tag. Spot-checked against the
   dev server — `view-source` shows both alternates inline.
5. **Page-level Server Component testing is new in this repo.** The
   addition of `app/**/*.test.tsx` to the vitest include is the
   first place we test an async Server Component by awaiting it
   directly. The pattern works for the listing page because its
   tree only depends on `listPublishedPosts` (which we mock) and a
   client-side `<Nav>` (which renders happily under happy-dom). If
   future Server Component tests touch `next/headers` or `cookies()`,
   they'll need to mock those too — flagged so the next FE dispatch
   doesn't bump into it cold.
6. **E2E spec not executed in this dispatch.** Same caveat as
   M3/M4/M5/M9-AEO. The spec compiles; Tester should run it under
   their seeded admin DB.
